J'ai pété les plombs sur la limite de contexte — voilà comment j'ai résolu ça

Si comme moi vous avez pété les plombs devant ce message... vous savez de quoi je parle.

Le problème : une progression géométrique de tokens

Utiliser Claude comme ChatGPT, c'est une mauvaise idée. Chaque message ajoute l'intégralité de la conversation précédente dans le contexte. Ce n'est pas une progression linéaire — c'est géométrique. Ajoutez un modèle premium et la facture explose.

Au bout de quelques heures de travail sur un projet complexe, vous touchez le plafond. Claude vous dit qu'il ne peut plus continuer. Tout le contexte est perdu. Vous repartez de zéro.

Ma solution : un skill de gestion de sessions

J'ai construit un skill qui crée une vraie notion de session dans Claude. Le principe est simple : vous sauvegardez quand vous voulez, vous rechargez dans une nouvelle tâche. Claude compresse votre contexte entre les deux — les informations essentielles sont distillées, le bruit éliminé.

Résultat : jusqu'à 90% de tokens en moins par rapport à une conversation continue.

Comment l'installer

Le skill est en téléchargement gratuit — lien ICI. Glissez le fichier téléchargé en pièce jointe dans Claude et demandez-lui d'exécuter le prompt. Vous n'aurez qu'à valider en faisant "enregistrer la compétence".

⚠️ Nécessite l'app desktop Claude, en mode Cowork avec un dossier connecté.
