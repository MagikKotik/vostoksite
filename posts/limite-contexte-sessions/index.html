J'ai pété les plombs sur la limite de contexte — voilà comment j'ai résolu ça

Si comme moi vous avez pété les plombs devant ce message... vous savez de quoi je parle.

Le problème : une progression géométrique de tokens

Utiliser Claude comme ChatGPT, c'est une mauvaise idée. Chaque message ajoute l'intégralité de la conversation précédente dans le contexte. Ce n'est pas une progression linéaire — c'est géométrique. Ajoutez un modèle premium et la facture explose.

Le plafond peut arriver en quelques minutes si vous rechargez une conversation ancienne — le contexte accumulé depuis des semaines est déjà là avant même que vous ayez dit bonjour. Claude vous dit qu'il ne peut plus continuer. Tout le contexte est perdu. Vous repartez de zéro.

Ma solution : un skill de gestion de sessions

J'ai construit un skill qui crée une vraie notion de session dans Claude. Le principe : vous sauvegardez quand vous voulez, vous rechargez dans une nouvelle tâche. Claude compresse votre contexte entre les deux — les informations essentielles sont distillées, le bruit éliminé.

Résultat : jusqu'à 90% de tokens en moins par rapport à une conversation continue.

Comment ça marche concrètement

Quand vous tapez /sessions save, Claude génère un fichier Markdown ultra-compressé dans votre dossier connecté — état du projet, décisions clés, prochaines étapes. Pas de roman : un résumé technique pensé pour être relu par Claude, pas par vous. Ce qui rend l'approche solide : chaque session repart propre, sans l'historique des essais et erreurs qui gonflent le contexte inutilement.

Quand vous tapez /sessions load <nom> dans une nouvelle conversation, Claude lit ce fichier, décompresse le contexte, et vous fait un briefing en français naturel : où on en était, ce qui a été décidé, ce qu'il reste à faire. En 30 secondes, vous êtes de retour dans le bain.

La commande /dir et les jokers

Vous pouvez avoir autant de sessions que vous voulez — une par projet, une par sujet. La commande /sessions dir liste toutes vos sessions actives. Et le vrai confort : le wildcard *. Vous ne vous souvenez plus du nom exact ? /sessions load projet* trouve toutes les sessions qui commencent par "projet". /sessions load *site* trouve tout ce qui contient "site". Claude charge ce qui correspond et vous briefera sur chacune.

Vous pouvez aussi charger plusieurs sessions en une commande — pratique pour fusionner deux projets liés. Et quand une session est terminée, /sessions save --close l'archive proprement : elle disparaît de vos listes sans être supprimée.

Ce que le fichier contient vraiment

Le fichier de session n'est pas lisible au sens humain du terme — c'est volontaire. Claude écrit dans un format télégraphique ultra-compressé en anglais, avec des symboles plutôt que des mots. Quelque chose comme :

## State
- site v2 ✓ deployed, perf ok
- login bug mobile ✗ → suspect cookie cfg
- choice: Tailwind ✓, Bootstrap ✗
## Next
- fix login bug
- ? add newsletter section

Un fichier de session tient généralement en 20 à 50 lignes, quelle que soit la durée de la conversation originale. C'est ce ratio qui explique les 90% de réduction.

Comment l'installer

Le skill est en téléchargement gratuit — lien ICI. Glissez le fichier téléchargé en pièce jointe dans Claude et demandez-lui d'exécuter le prompt. Vous n'aurez qu'à valider en faisant "enregistrer la compétence".

⚠️ Nécessite l'app desktop Claude, en mode Cowork avec un dossier connecté.
